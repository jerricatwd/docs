## Overview

This tutorial helps users troubleshoot common issues that may prevent them from successfully logging into the Dashboard. It covers credential verification, domain formatting, password reset steps, session management, and advanced troubleshooting methods.

## Prerequisites

Before starting, users should have:

Dashboard login email

Current password (if available)

Access to email inbox for password reset

A stable internet connection

## Step 1 — Verify Login Credentials

Ensure that all required information is entered accurately.
Confirm that the website field contains the correct domain URL.
Check that your email address is typed correctly.
For accuracy, it is recommended to copy and paste the provided password.

## Step 2 — Remove the WWW from the Domain URL

Make sure to remove the WWW part from the URL and use the format:
https://yourdomain.com

## Step 4 — Session and Browser Management

If you remain logged in for long periods, session issues can occur.
Log out and log back in periodically.
Clear your browser cache regularly to help prevent login problems.

## Advanced Troubleshooting

<Accordion title="Having issues viewing the dashboard?">
Try accessing the Dashboard using a different web browser.
</Accordion>
<Accordion title="Constantly loading?">
Test your internet connection by switching to mobile data or another network.
</Accordion>
<Accordion title="Dashboard isn't loading at all">
Some antivirus programs may block access to the Dashboard. If you suspect this, contact your IT team to ensure the Dashboard is not being blocked or quarantined.
</Accordion>

## Conclusion

By verifying credentials, correcting the domain format, resetting your password, managing browser sessions, and checking for security restrictions, most Dashboard login issues can be resolved quickly and easily. If problems persist, reach out to support for further assistance.
